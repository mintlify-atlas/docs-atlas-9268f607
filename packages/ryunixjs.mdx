## Overview

`@unsetsoft/ryunixjs` is the foundational engine powering the RyunixJS framework. It provides a lightweight, high-performance library responsible for Virtual DOM reconciliation, hooks-based state management, concurrent rendering, and comprehensive Server-Side Rendering capabilities.

## Core API

### Rendering

#### render

Renders a Ryunix element into a DOM container.

```javascript
import { render, createElement } from '@unsetsoft/ryunixjs'

function App() {
  return createElement('div', null, 'Hello, RyunixJS!')
}

render(createElement(App), document.getElementById('root'))
```

**Signature:**
```typescript
render(element: RyunixElement, container: HTMLElement): void
```

#### hydrate

Hydrates server-rendered HTML with client-side interactivity.

```javascript
import { hydrate, createElement } from '@unsetsoft/ryunixjs'

hydrate(createElement(App), document.getElementById('root'))
```

**Signature:**
```typescript
hydrate(element: RyunixElement, container: HTMLElement): void
```

<Tip>
Use `hydrate` when your HTML was pre-rendered on the server to avoid re-rendering and preserve existing DOM nodes.
</Tip>

#### hydrateIslands

Selectively hydrates specific components (Islands Architecture).

```javascript
import { hydrateIslands } from '@unsetsoft/ryunixjs'

hydrateIslands()
```

#### init

Initializes the Ryunix state. Called automatically by `render`.

```javascript
import { init } from '@unsetsoft/ryunixjs'

init()
```

### Element Creation

#### createElement

Creates a Virtual DOM element.

```javascript
import { createElement } from '@unsetsoft/ryunixjs'

const element = createElement(
  'button',
  { onClick: () => console.log('clicked') },
  'Click me'
)
```

**Signature:**
```typescript
createElement(
  type: string | Function,
  props: object | null,
  ...children: any[]
): RyunixElement
```

#### Fragment

Groups multiple children without adding extra DOM nodes.

```javascript
import { createElement, Fragment } from '@unsetsoft/ryunixjs'

function List() {
  return createElement(Fragment, null,
    createElement('li', null, 'Item 1'),
    createElement('li', null, 'Item 2')
  )
}
```

#### cloneElement

Clones an element with new props.

```javascript
import { cloneElement } from '@unsetsoft/ryunixjs'

const cloned = cloneElement(element, { newProp: 'value' })
```

#### isValidElement

Checks if a value is a valid Ryunix element.

```javascript
import { isValidElement } from '@unsetsoft/ryunixjs'

if (isValidElement(value)) {
  // Handle element
}
```

## Hooks System

RyunixJS provides a comprehensive hooks API for managing state, side effects, and component logic.

### State Management

#### useStore

Manages component state (equivalent to React's `useState`).

```javascript
import { useStore } from '@unsetsoft/ryunixjs'

function Counter() {
  const [count, setCount] = useStore(0)
  
  return createElement('button',
    { onClick: () => setCount(count + 1) },
    `Count: ${count}`
  )
}
```

**Signature:**
```typescript
useStore<T>(initialState: T | (() => T)): [T, (newState: T | ((prev: T) => T)) => void]
```

<Note>
State updates can accept either a new value or a function that receives the previous state and returns the new state.
</Note>

#### useReducer

Manages complex state logic with a reducer function.

```javascript
import { useReducer } from '@unsetsoft/ryunixjs'

function reducer(state, action) {
  switch (action.type) {
    case 'increment': return { count: state.count + 1 }
    case 'decrement': return { count: state.count - 1 }
    default: return state
  }
}

function Counter() {
  const [state, dispatch] = useReducer(reducer, { count: 0 })
  
  return createElement('div', null,
    createElement('p', null, `Count: ${state.count}`),
    createElement('button', 
      { onClick: () => dispatch({ type: 'increment' }) },
      'Increment'
    )
  )
}
```

**Signature:**
```typescript
useReducer<S, A>(
  reducer: (state: S, action: A) => S,
  initialState: S,
  init?: (initial: S) => S
): [S, (action: A) => void]
```

#### usePersistentStore

State that automatically syncs with `localStorage`.

```javascript
import { usePersistentStore } from '@unsetsoft/ryunixjs'

function Settings() {
  const [theme, setTheme] = usePersistentStore('theme', 'dark')
  
  return createElement('button',
    { onClick: () => setTheme(theme === 'dark' ? 'light' : 'dark') },
    `Theme: ${theme}`
  )
}
```

**Signature:**
```typescript
usePersistentStore<T>(key: string, initialState: T): [T, (value: T) => void]
```

#### useSwitch

Optimized boolean toggle state.

```javascript
import { useSwitch } from '@unsetsoft/ryunixjs'

function Toggle() {
  const [isOn, toggle] = useSwitch(false)
  
  return createElement('button',
    { onClick: toggle },
    isOn ? 'ON' : 'OFF'
  )
}
```

### Side Effects

#### useEffect

Performs side effects after render.

```javascript
import { useEffect, useStore } from '@unsetsoft/ryunixjs'

function DataFetcher() {
  const [data, setData] = useStore(null)
  
  useEffect(() => {
    fetch('/api/data')
      .then(res => res.json())
      .then(setData)
    
    return () => {
      // Cleanup function (optional)
    }
  }, []) // Empty array = run once on mount
  
  return createElement('div', null, JSON.stringify(data))
}
```

**Signature:**
```typescript
useEffect(callback: () => void | (() => void), deps?: any[]): void
```

<Warning>
Omitting the dependencies array causes the effect to run after every render. Always specify dependencies to avoid performance issues.
</Warning>

#### useLayoutEffect

Synchronous effect that runs before browser paint. Use for DOM measurements.

```javascript
import { useLayoutEffect, useRef } from '@unsetsoft/ryunixjs'

function MeasureElement() {
  const ref = useRef(null)
  
  useLayoutEffect(() => {
    const height = ref.current.offsetHeight
    console.log('Element height:', height)
  }, [])
  
  return createElement('div', { ref }, 'Measure me')
}
```

### Memoization

#### useMemo

Memoizes computed values.

```javascript
import { useMemo, useStore } from '@unsetsoft/ryunixjs'

function ExpensiveCalculation() {
  const [count, setCount] = useStore(0)
  
  const expensiveValue = useMemo(() => {
    console.log('Computing...')
    return count * 2
  }, [count])
  
  return createElement('div', null, expensiveValue)
}
```

#### useCallback

Memoizes callback functions.

```javascript
import { useCallback, useStore } from '@unsetsoft/ryunixjs'

function Parent() {
  const [count, setCount] = useStore(0)
  
  const handleClick = useCallback(() => {
    setCount(c => c + 1)
  }, [])
  
  return createElement(Child, { onClick: handleClick })
}
```

### Refs

#### useRef

Creates a mutable reference that persists across renders.

```javascript
import { useRef } from '@unsetsoft/ryunixjs'

function TextInput() {
  const inputRef = useRef(null)
  
  const focusInput = () => {
    inputRef.current.focus()
  }
  
  return createElement('div', null,
    createElement('input', { ref: inputRef }),
    createElement('button', { onClick: focusInput }, 'Focus')
  )
}
```

### Utility Hooks

#### useId

Generates stable, unique IDs for accessibility (SSR-safe).

```javascript
import { useId } from '@unsetsoft/ryunixjs'

function FormField() {
  const id = useId()
  
  return createElement('div', null,
    createElement('label', { htmlFor: id }, 'Name:'),
    createElement('input', { id })
  )
}
```

#### useDebounce

Debounces a value update.

```javascript
import { useDebounce, useStore } from '@unsetsoft/ryunixjs'

function SearchInput() {
  const [value, setValue] = useStore('')
  const debouncedValue = useDebounce(value, 500)
  
  // debouncedValue updates 500ms after user stops typing
  
  return createElement('input', {
    value,
    onInput: (e) => setValue(e.target.value)
  })
}
```

#### useThrottle

Throttles value updates.

```javascript
import { useThrottle, useStore } from '@unsetsoft/ryunixjs'

function ScrollPosition() {
  const [scrollY, setScrollY] = useStore(0)
  const throttledScroll = useThrottle(scrollY, 300)
  
  // throttledScroll updates at most once per 300ms
  
  return createElement('p', null, `Scroll: ${throttledScroll}`)
}
```

#### useMetadata

Manages document metadata dynamically.

```javascript

function PageComponent() {

    canonical: 'https://example.com/page'

    }

}
```

### Concurrent Features

#### useTransition

Marks updates as non-urgent transitions.

```javascript
import { useTransition, useStore } from '@unsetsoft/ryunixjs'

function SearchResults() {
  const [query, setQuery] = useStore('')
  const [isPending, startTransition] = useTransition()
  
  const handleChange = (value) => {
    setQuery(value)
    startTransition(() => {
      // Heavy computation or state update
      performExpensiveSearch(value)
    })
  }
  
  return createElement('div', null,
    createElement('input', { onInput: (e) => handleChange(e.target.value) }),
    isPending ? 'Loading...' : 'Results'
  )
}
```

#### useDeferredValue

Defers a value update to keep UI responsive.

```javascript
import { useDeferredValue, useStore } from '@unsetsoft/ryunixjs'

function FilteredList({ items }) {
  const [filter, setFilter] = useStore('')
  const deferredFilter = useDeferredValue(filter)
  
  // Use deferredFilter for expensive operations
  const filtered = items.filter(item => 
    item.includes(deferredFilter)
  )
  
  return createElement('div', null,
    createElement('input', { 
      value: filter,
      onInput: (e) => setFilter(e.target.value) 
    }),
    filtered.map(item => createElement('p', { key: item }, item))
  )
}
```

## Advanced Components

### memo

Prevents unnecessary re-renders.

```javascript
import { memo } from '@unsetsoft/ryunixjs'

const ExpensiveComponent = memo(function ExpensiveComponent({ data }) {
  console.log('Rendering expensive component')
  return createElement('div', null, data)
})
```

### lazy

Code-splits components.

```javascript

    createElement(LazyComponent)
  )
}
```

### Complete Counter App

```javascript
import { render, createElement, useStore } from '@unsetsoft/ryunixjs'

function Counter() {
  const [count, setCount] = useStore(0)
```

### Data Fetching with Effects

```javascript
import { render, createElement, useStore, useEffect } from '@unsetsoft/ryunixjs'

function UserProfile({ userId }) {
  const [user, setUser] = useStore(null)
  const [loading, setLoading] = useStore(true)
  const [error, setError] = useStore(null)
  
  useEffect(() => {
    let cancelled = false
    
    setLoading(true)
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(data => {
        if (!cancelled) {
          setUser(data)
          setLoading(false)
        }
      })
      .catch(err => {
        if (!cancelled) {
          setError(err.message)
          setLoading(false)
        }
      })
    
    return () => {
      cancelled = true
    }
  }, [userId])
  
  if (loading) return createElement('div', null, 'Loading...')
  if (error) return createElement('div', null, `Error: ${error}`)
  if (!user) return createElement('div', null, 'No user found')
  
  return createElement('div', null,
    createElement('h1', null, user.name),
    createElement('p', null, user.email)
  )
}
```

## Performance Tips

<Steps>
  <Step title="Use memo for expensive components">
    Wrap components that perform heavy computations with `memo` to prevent unnecessary re-renders.
  </Step>
  
  <Step title="Optimize dependency arrays">
    Always specify accurate dependency arrays in `useEffect`, `useMemo`, and `useCallback`.
  </Step>
  
  <Step title="Lazy load heavy components">
    Use `lazy` and `Suspense` to split large components into separate bundles.
  </Step>
  
  <Step title="Batch state updates">
    Use `batchUpdates` when updating multiple state variables simultaneously.
  </Step>
  
  <Step title="Use transitions for heavy updates">
    Mark non-urgent updates with `useTransition` to keep the UI responsive.
  </Step>
</Steps>
