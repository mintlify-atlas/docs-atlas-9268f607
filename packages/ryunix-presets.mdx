## Overview

`@unsetsoft/ryunix-presets` is the central package managing the build system of RyunixJS. It abstracts the complexity of Webpack, Babel, and specialized loaders, providing a unified and optimized development experience with zero-config defaults and extensive customization options.

  <Card title="Full Control" icon="sliders">

  <Card title="SSR Support" icon="server">

  </Card>

## Installation

```bash
npm install @unsetsoft/ryunix-presets
```

<Note>
This package requires `@unsetsoft/ryunixjs` as a peer dependency (^1.2.4-canary.0 or ^1.2.4).
</Note>

## Configuration

RyunixJS projects are configured via a `ryunix.config.js` (or `.cjs`) file at the project root.

### Basic Configuration

```javascript
// ryunix.config.js
module.exports = {
  ssr: true,
  mdx: false,
  rootDir: 'src',
  buildDir: '.ryunix',
  port: 3000,
  favicon: true,
  debug: false
}
```

### Configuration Options

#### Core Options

| Option | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `ssr` | `boolean` | `true` | Enables Server-Side Rendering compilation |
| `mdx` | `boolean` | `false` | Enables native MDX support with loaders |
| `rootDir` | `string` | `"src"` | Source code directory |
| `buildDir` | `string` | `".ryunix"` | Output directory for builds |
| `port` | `number` | `3000` | Development server port |
| `favicon` | `string \| boolean` | `true` | Path to favicon or boolean to enable default |
| `debug` | `boolean` | `false` | Enable verbose logging |
| `env` | `object` | `{}` | Environment variables to inject |
| `proxy` | `array \| object` | - | Proxy configuration for dev server |

### Server Configuration

Configure CORS and Content Security Policy:

```javascript
module.exports = {
  server: {
    csp: true, // or custom CSP string
    cors: {
      enabled: true,
      origin: '*',
      methods: 'GET,POST,PUT,DELETE',
      headers: 'Content-Type,Authorization',
      credentials: false
    }
  }
}
```

#### Server Options

| Option | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `server.csp` | `boolean \| string` | `false` | Content Security Policy |
| `server.cors.enabled` | `boolean` | `false` | Enable CORS |
| `server.cors.origin` | `string` | `'*'` | Allowed origins |
| `server.cors.methods` | `string` | - | Allowed HTTP methods |
| `server.cors.headers` | `string` | - | Allowed headers |
| `server.cors.credentials` | `boolean` | `false` | Allow credentials |

### ESLint Configuration

Customize ESLint behavior during development:

```javascript
module.exports = {
  eslint: {
    files: ['src/**/*.{js,jsx,ryx}'],
    plugins: {
      // Custom ESLint plugins
    },
    rules: {
      'no-console': 'warn',
      'no-unused-vars': 'error',
      'ryunix/no-unsafe-render': 'warn'
    }
  }
}
```

#### ESLint Options

| Option | Type | Description |
| :--- | :--- | :--- |
| `eslint.files` | `string[]` | File patterns to lint |
| `eslint.plugins` | `object` | Custom ESLint plugins |
| `eslint.rules` | `object` | ESLint rules configuration |

### Webpack Customization

Extend the underlying Webpack configuration:

```javascript
const path = require('path')

module.exports = {
  webpack: {
    resolve: {
      alias: {
        '@components': path.resolve(__dirname, 'src/components'),
        '@utils': path.resolve(__dirname, 'src/utils'),
        '@styles': path.resolve(__dirname, 'src/styles')
      },
      extensions: ['.js', '.jsx', '.ryx', '.json']
    },
    plugins: [
      // Add custom Webpack plugins
    ],
    module: {
      rules: [
        // Add custom loaders
        {
          test: /\.custom$/,
          use: 'custom-loader'
        }
      ]
    },
    experiments: {
      lazyCompilation: true // Enable lazy compilation
    }
  }
}
```

#### Webpack Options

| Option | Type | Description |
| :--- | :--- | :--- |
| `webpack.resolve.alias` | `object` | Path aliases for imports |
| `webpack.resolve.extensions` | `string[]` | File extensions to resolve |
| `webpack.resolve.fallback` | `object` | Polyfills for Node.js modules |
| `webpack.plugins` | `array` | Custom Webpack plugins |
| `webpack.module.rules` | `array` | Custom loader rules |
| `webpack.devServer.allowedHosts` | `string \| array` | Allowed hosts for dev server |
| `webpack.externals` | `array \| object` | External dependencies |
| `webpack.experiments` | `object` | Experimental Webpack features |

### Environment Variables

Inject environment variables at build time:

```javascript
module.exports = {
  env: {
    API_URL: process.env.API_URL || 'https://api.example.com',
    FEATURE_FLAG: true
  }
}
```

Access in your code:

```javascript
const apiUrl = process.env.API_URL
```

<Tip>
Environment variables are replaced at build time using `dotenv-webpack`. Create a `.env` file for local development.
</Tip>

### Proxy Configuration

Proxy API requests during development:

```javascript
module.exports = {
  proxy: {
    '/api': {
      target: 'http://localhost:4000',
      changeOrigin: true,
      pathRewrite: { '^/api': '' }
    }
  }
}
```

Or use array format:

```javascript
module.exports = {
  proxy: [
    {
      context: ['/api', '/auth'],
      target: 'http://localhost:4000'
    }
  ]
}
```

For backwards compatibility, legacy options are supported:

## Build Pipelines

The presets package manages three main compilation pipelines:

### 1. Client Pipeline

**Purpose:** Compiles assets for the browser

**Features:**
- Code splitting and lazy loading
- Hot Module Replacement (HMR)
- CSS extraction and optimization
- Asset optimization (images, fonts)
- Source maps for debugging

**Output:** `.ryunix/client/`

### 2. Server Pipeline

**Purpose:** Compiles server-side logic for Node.js environments

**Features:**
- SSR renderer compilation
- Server-only code optimization
- Node.js module resolution
- Environment variable injection

**Output:** `.ryunix/server/`

### 3. SSG/Prerender Pipeline

**Purpose:** Generates static HTML during build phase

**Features:**
- Static site generation
- Pre-rendering for SEO
- Sitemap generation
- Meta tag injection

**Output:** `.ryunix/static/`

## Loader Configurations

The presets package includes specialized loaders for different file types:

### JavaScript/JSX/RYX Files

```javascript
// Automatically configured
{
  test: /\.(js|jsx|ryx)$/,
  exclude: /node_modules/,
  use: {
    loader: 'babel-loader',
    options: {
      presets: [
        '@babel/preset-env',
        ['@babel/preset-react', {
          runtime: 'automatic',
          importSource: '@unsetsoft/ryunixjs'
        }]
      ]
    }
  }
}
```

### CSS/SASS/SCSS Files

```javascript
// CSS
{
  test: /\.css$/,
  use: [
    isDev ? 'style-loader' : MiniCssExtractPlugin.loader,
    'css-loader',
    'postcss-loader'
  ]
}

// SASS/SCSS
{
  test: /\.(sass|scss)$/,
  use: [
    isDev ? 'style-loader' : MiniCssExtractPlugin.loader,
    'css-loader',
    'postcss-loader',
    'sass-loader'
  ]
}
```

### MDX Files

```javascript
// When mdx: true in config
{
  test: /\.mdx$/,
  use: [
    'babel-loader',
    {
      loader: '@mdx-js/loader',
      options: {
        remarkPlugins: [
          remarkGfm,
          remarkFrontmatter,
          remarkMdxFrontmatter
        ],
        rehypePlugins: [rehypeHighlight]
      }
    }
  ]
}
```

### Asset Files

```javascript
// Images
{
  test: /\.(png|jpg|jpeg|gif|svg|webp)$/,
  type: 'asset/resource'
}

// Fonts
{
  test: /\.(woff|woff2|eot|ttf|otf)$/,
  type: 'asset/resource'
}
```

## Development Server

The development server provides:

- **Hot Module Replacement (HMR)**: Instant updates without page refresh
- **Live Reloading**: Automatic browser refresh on file changes
- **Error Overlay**: Visual error display in the browser
- **Proxy Support**: API request proxying
- **HTTPS Support**: Optional SSL certificates
- **History API Fallback**: SPA routing support

### Server Configuration

```javascript
module.exports = {
  port: 3000,
  webpack: {
    devServer: {
      allowedHosts: 'all', // or ['localhost', '.example.com']
      hot: true,
      compress: true,
      historyApiFallback: true
    }
  }
}
```

## CLI Reference

### Binary: `ryunix`

Location: `./webpack/bin/index.mjs`

### Available Commands

#### `ryunix dev`

Starts the development server:

```bash
ryunix dev

# Custom port
ryunix dev --port 4000

# Enable debugging
ryunix dev --debug
```

**Options:**
- `--port <number>`: Override dev server port
- `--debug`: Enable verbose logging
- `--no-open`: Don't open browser automatically

#### `ryunix build`

Builds for production:

```bash
ryunix build

# With bundle analysis
ryunix build --analyze

# Skip SSR build
ryunix build --no-ssr
```

**Options:**
- `--analyze`: Generate bundle analysis report
- `--no-ssr`: Skip server-side rendering build
- `--profile`: Enable profiling mode

## Advanced Usage

### Custom Webpack Plugin

```javascript
// ryunix.config.js
const CustomPlugin = require('./plugins/custom-plugin')

module.exports = {
  webpack: {
    plugins: [
      new CustomPlugin({
        option: 'value'
      })
    ]
  }
}
```

### Multi-Environment Configuration

```javascript
// ryunix.config.js
const isDev = process.env.NODE_ENV !== 'production'

module.exports = {
  debug: isDev,
  port: isDev ? 3000 : 8080,
  webpack: {
    devServer: {
      allowedHosts: isDev ? 'all' : ['example.com']
    }
  }
}
```

### PostCSS Configuration

Create `postcss.config.js`:

```javascript
module.exports = {
  plugins: [
    require('autoprefixer'),
    require('postcss-nested'),
    require('postcss-custom-properties')
  ]
}
```

### Babel Configuration

Create `.babelrc` or `babel.config.js`:

```javascript
module.exports = {
  presets: [
    ['@babel/preset-env', {
      targets: '> 0.25%, not dead'
    }]
  ],
  plugins: [
    '@babel/plugin-proposal-optional-chaining',
    '@babel/plugin-proposal-nullish-coalescing-operator'
  ]
}
```

## Troubleshooting

### Build Performance

If builds are slow, enable lazy compilation:

```javascript
module.exports = {
  webpack: {
    experiments: {
      lazyCompilation: true
    }
  }
}
```

### Memory Issues

Increase Node.js memory limit:

```json
{
  "scripts": {
    "build": "NODE_OPTIONS='--max-old-space-size=4096' ryunix build"
  }
}
```

### ESLint Errors

Disable ESLint during build:

```javascript
module.exports = {
  eslint: false
}
```

### Hot Reload Not Working

Check firewall settings or try:

```javascript
module.exports = {
  webpack: {
    devServer: {
      allowedHosts: 'all'
    }
  }
}
```
