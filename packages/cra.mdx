## Overview

`@unsetsoft/cra` (Create Ryunix App) is the official scaffolding tool for RyunixJS applications. It provides an interactive CLI experience that guides you through creating a new project with optimized defaults, modern tooling, and optional integrations like Tailwind CSS and ESLint.

## Installation

No installation required! Use `npx` to run the latest version:

```bash
npx @unsetsoft/cra@latest
```

### Global Installation (Optional)

```bash
npm install -g @unsetsoft/cra
cra my-app
```

## Quick Start

### Basic Usage

```bash
npx @unsetsoft/cra@latest
```

The CLI will prompt you for:
1. Project name
2. Release channel (Latest or Canary)
3. Optional integrations (Tailwind, ESLint, VS Code)

### With Project Name

```bash
npx @unsetsoft/cra@latest my-ryunix-app
```

### With Flags

```bash
npx @unsetsoft/cra@latest my-app --canary --tailwind --eslint --vscode
```

## Interactive Experience

The CLI provides a guided setup process:

<Steps>
  <Step title="Project Name">
    Enter the directory name for your new application.
    
    ```bash
    ? Project name: my-ryunix-app
    ```
  </Step>
  
  <Step title="Release Channel">
    Choose between **Latest** (stable) or **Canary** (cutting-edge features).
    
    ```bash

      Canary (experimental)
    ```

  <Step title="Tailwind CSS">
    Optional automatic initialization of Tailwind CSS.
    
    ```bash
    ? Initialize Tailwind CSS? (y/N)
    ```

  </Step>
  
  <Step title="ESLint">
    Pre-configured linting rules optimized for Ryunix.
    
    ```bash
    ? Initialize ESLint? (y/N)
    ```
  </Step>
  
  <Step title="VS Code Integration">
    Automatic workspace settings for Ryunix VS Code Extension.
    
    ```bash
    ? Add VS Code settings? (y/N)
    ```
  </Step>
</Steps>

## Command Line Flags

### Version and Help

| Flag | Description |
| :--- | :--- |
| `-v, --version` | Output the current version of the CLI |
| `-h, --help` | Display help message with usage instructions |

**Example:**
```bash
npx @unsetsoft/cra@latest --version
# Output: 1.1.20-canary.4
```

### Release Channel

| Flag | Description |
| :--- | :--- |
| `--canary` | Use Canary channel for Ryunix dependencies |
| `--latest` | Use Latest channel (default) |

**Example:**
```bash
npx @unsetsoft/cra@latest my-app --canary
```

<Note>
Canary releases include experimental features and may have breaking changes. Recommended for testing new features or contributing to development.
</Note>

### Optional Integrations

| Flag | Description |
| :--- | :--- |
| `--tailwind` | Initialize with Tailwind CSS configuration |
| `--eslint` | Initialize with ESLint configuration |
| `--vscode` | Add VS Code settings for Ryunix extension |

**Example:**
```bash
npx @unsetsoft/cra@latest my-app --tailwind --eslint
```

## Project Structure

The CLI generates a complete project with the following structure:

```
my-ryunix-app/
├── src/
│   ├── app/
│   │   └── App.ryx          # Main application component
│   ├── components/          # Reusable components
│   └── index.js             # Application entry point
├── public/
│   ├── index.html           # HTML template
│   └── favicon.ico          # Default favicon
├── .gitignore               # Git ignore rules
├── package.json             # Dependencies and scripts
├── ryunix.config.js         # Ryunix configuration
└── README.md                # Project documentation
```

### With Optional Integrations

#### Tailwind CSS

```
├── tailwind.config.js       # Tailwind configuration
├── postcss.config.js        # PostCSS configuration
└── src/
    └── styles/
        └── globals.css      # Global styles with Tailwind
```

#### ESLint

```
├── .eslintrc.js             # ESLint configuration
└── .eslintignore            # ESLint ignore rules
```

#### VS Code

```
├── .vscode/
│   └── settings.json        # VS Code workspace settings
│   └── extensions.json      # Recommended extensions
```

```json
  "private": true,
```

### ryunix.config.js

```javascript

  ssr: true,
  mdx: false,
  rootDir: 'src',

  favicon: true

  }
```

## Available Scripts

The generated project includes these npm scripts:

### `npm run dev`

Starts the development server with hot reload.

```bash
npm run dev
```

- Opens [http://localhost:3000](http://localhost:3000)
- Hot Module Replacement enabled
- Error overlay for debugging
- Fast refresh on file changes

### `npm run build`

Builds the app for production.

```bash
npm run build
```

- Optimized production build
- Minified JavaScript and CSS
- Source maps for debugging
- Output to `.ryunix/` directory

Alias for `npm run dev`.


## Troubleshooting

### Port Already in Use

If port 3000 is busy, specify a different port:

```bash
# Edit ryunix.config.js
module.exports = {
  port: 3001
}
```

### Installation Fails

Ensure you're using a supported Node.js version:

```bash
node --version
# Should be 20.x, 22.x, or 24.x
```

### Permission Errors

On Linux/Mac, you might need to fix npm permissions:

```bash
sudo chown -R $(whoami) ~/.npm
```

### Network Issues

If installation is slow, try a different registry:

```bash
npm config set registry https://registry.npmjs.org/
```

## Advanced Usage

### Programmatic API

You can use CRA programmatically:

```javascript
const { exec } = require('child_process')
const path = require('path')

const projectName = 'my-app'
const cliPath = require.resolve('@unsetsoft/cra')

exec(`node ${cliPath} ${projectName} --canary`, (error, stdout) => {
  if (error) {
    console.error('Error creating app:', error)
    return
  }
  console.log(stdout)
})
```

### Environment Variables

Configure behavior with environment variables:

```bash
# Skip prompts (use defaults)
CRA_SKIP_PROMPTS=true npx @unsetsoft/cra@latest my-app

# Custom template path
CRA_TEMPLATE_PATH=/path/to/template npx @unsetsoft/cra@latest my-app
```
