## Overview

`batchUpdates` allows you to batch multiple state updates together, ensuring they're processed in a single render cycle. This improves performance by avoiding unnecessary re-renders when updating multiple pieces of state.

## Import

```javascript
import { batchUpdates } from 'ryunix'
```

## Syntax

```javascript
let isBatching = false
let pendingUpdates = []

const batchUpdates = (callback) => {
  const wasBatching = isBatching
  isBatching = true

  try {
    callback()
  } finally {
    isBatching = wasBatching

    if (!isBatching && pendingUpdates.length > 0) {
      flushUpdates()
    }
  }
}
```

## Parameters

<ParamField path="callback" type="Function" required>
  A function containing multiple state updates that should be batched together. All updates within this function will be processed in a single render cycle.
</ParamField>

## Internal Mechanism

### Queue System

```javascript
let isBatching = false
let pendingUpdates = []

const queueUpdate = (update) => {
  pendingUpdates.push(update)

  if (!isBatching) {
    flushUpdates()
  }
}

const flushUpdates = () => {
  if (pendingUpdates.length === 0) return

  const updates = pendingUpdates
  pendingUpdates = []

  // Execute all updates
  updates.forEach((update) => update())
}
```

### Batching Flow

1. **Enter Batch**: Set `isBatching = true`
2. **Execute Callback**: All updates are queued, not executed
3. **Exit Batch**: Set `isBatching = false`
4. **Flush Updates**: Execute all queued updates at once

## Advanced Patterns

### Nested Batching

```jsx
import { batchUpdates, useStore } from 'ryunix'

const Component = () => {
  const [state, setState] = useStore(initialState)
  
  const complexUpdate = () => {
    batchUpdates(() => {
      setState({ ...state, a: 1 })
      
      // Nested batch is safe
      batchUpdates(() => {
        setState({ ...state, b: 2 })
        setState({ ...state, c: 3 })
      })
      
      setState({ ...state, d: 4 })
    })
    // Still only 1 render
  }
}
```

### Conditional Batching

```jsx
import { batchUpdates, useStore } from 'ryunix'

const Component = ({ shouldBatch }) => {
  const [state, setState] = useStore({})
  
  const update = () => {
    const updates = () => {
      setState({ a: 1 })
      setState({ b: 2 })
      setState({ c: 3 })
    }
    
    if (shouldBatch) {
      batchUpdates(updates)
    } else {
      updates()
    }
  }
}
```

### Custom Batch Hook

```jsx
import { batchUpdates, useStore } from 'ryunix'

const useBatchedState = (initialState) => {
  const [state, setState] = useStore(initialState)
  
  const batchedSetState = (updates) => {
    batchUpdates(() => {
      if (typeof updates === 'function') {
        setState(updates)
      } else {
        setState({ ...state, ...updates })
      }
    })
  }
  
  return [state, batchedSetState]
}

// Usage
const Component = () => {
  const [state, setState] = useBatchedState({ a: 0, b: 0, c: 0 })
  
  const update = () => {
    setState({ a: 1, b: 2, c: 3 }) // Automatically batched
  }
}
```

## When to Use

<Tip>
  Use `batchUpdates` when:
  - Updating multiple related state values at once
  - Performing batch operations on data structures
  - Handling form submissions with multiple fields
  - Synchronizing multiple pieces of state after async operations
</Tip>

<Note>
  RyunixJS automatically batches updates in event handlers, so you may not need explicit batching in those cases.
</Note>

## Best Practices

<Tip>
  Group related state updates together in a single `batchUpdates` call for optimal performance.
</Tip>

<Tip>
  Use `batchUpdates` for state synchronization after data fetching to avoid intermediate loading states.
</Tip>

<Warning>
  Don't overuse batching. If updates aren't related or don't happen together, batching may hide bugs or make code harder to understand.
</Warning>

## Error Handling

The `batchUpdates` function uses try-finally to ensure proper cleanup:

```javascript
try {
  callback()
} finally {
  isBatching = wasBatching
  
  if (!isBatching && pendingUpdates.length > 0) {
    flushUpdates()
  }
}
```

Even if the callback throws an error, the batching state is properly restored and pending updates are flushed.

## Common Pitfalls

### Reading State During Batch

```jsx
// ❌ Bad - State may not reflect updates yet
batchUpdates(() => {
  setCount(count + 1)
  console.log(count) // Still shows old value
  setCount(count + 1) // Uses old value
})

// ✅ Good - Use functional updates
batchUpdates(() => {
  setCount(prev => prev + 1)
  setCount(prev => prev + 1) // Uses updated value
})
```

### Forgetting to Batch Async Updates

```jsx
// ❌ Bad - Updates outside batch
const fetchData = async () => {
  const result = await api.getData()
  setData(result)  // Separate render
  setLoading(false) // Separate render
}

// ✅ Good - Batch async results
const fetchData = async () => {
  const result = await api.getData()
  batchUpdates(() => {
    setData(result)
    setLoading(false)
  })
}
```
